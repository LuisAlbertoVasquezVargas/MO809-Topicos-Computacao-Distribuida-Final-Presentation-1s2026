\section{Introduction \& Motivation}

\begin{frame}{Introduction \& Motivation}
    \begin{itemize}
        \item \textbf{The Goal:} Discovering frequent itemsets (Association Rules) in massive transactional datasets.
        \item \textbf{The Barrier: Combinatorial Explosion}
        \begin{itemize}
            \item For a set of $n$ items, the search space is $O(2^n)$.
            \item If $n=1000$: $2^{1000}$ is a number larger than the atoms in the observable universe.
            \item \textbf{Reality Check:} Checking every combination is mathematically impossible, even with the world's most powerful clusters.
        \end{itemize}
    \end{itemize}
\end{frame}

\begin{frame}{Introduction \& Motivation}
    \begin{itemize}
        \item \textbf{Hardware Evolution:} 
        \begin{itemize}
            \item We have the ``Ferrari'' of computing: \textbf{Apache Spark RDDs} (In-memory, high-speed distributed parallelism).
        \end{itemize}
        \vfill
        \item \textbf{Software Bottleneck:} 
        \begin{itemize}
            \item Even a Ferrari is slow if it is stuck in a \textbf{traffic jam} of inefficient data structures.
            \item Legacy structures (Hash Trees) create a ``bottleneck'' that wastes the speed of RAM.
        \end{itemize}
    \end{itemize}
    \vfill
    \begin{alertblock}{The Core Thesis}
        Infrastructure alone cannot solve $O(2^n)$. To exploit the speed of Spark, we must replace ``Disk-era'' structures with memory-optimized \textbf{Tries}.
    \end{alertblock}
\end{frame}
